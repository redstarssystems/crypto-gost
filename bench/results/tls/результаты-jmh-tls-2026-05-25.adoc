== Streebog-256

Streebog256Bench **не завершился** за отведённое время (~90 мин).
JMH не смог завершить 3 fork'а для всех трёх размеров входа (32 B, 1 KB, 16 KB)
в рамках общего прогона. Для получения данных по Streebog требуется
отдельный запуск:

[source,bash]
----
java -jar target/benchmarks.jar Streebog256Bench -wi 5 -i 5 -f 3
----

Предыдущие цифры (2026-05-15) актуальны: 78,7 MB/s на 16 KB.

== Выводы

. **Код не регрессировал**. Все показатели в пределах шума относительно
  прогона 2026-05-15. Единственное исключение — `ECDHE shared secret`
  (+12,4 %), вероятно связанное с JDK 25.0.2 → 25.0.3 (JIT-улучшения
  для point multiplication).

. **Валидация сертификатов не является узким местом.** `NoVerifyHandshakeBench`
  показал, что без cert-validation handshake выполняется за 9,4 ms
  против 11,3 ms с валидацией (разница ~2 ms или 16 %). Чистая cert
  validation (`CertValidationBench`) занимает ~3,7 µs — на порядки
  меньше асимметричной криптографии.

. **Асимметричная криптография — доминирующий фактор.** 81 % времени
  handshake уходит на ECDHE + sign/verify. Для существенного ускорения
  требуется оптимизация point multiplication (нативная реализация).

. **Record layer — без изменений.** 50,8 MB/s protect, 49,5 MB/s unprotect.
  Арифметика Кузнечика-MGM на текущем уровне оптимизации стабильна.

. **Streebog не допрогнан.** Для завершения требуется отдельный запуск
  Streebog256Bench (чтобы не растягивать общий прогон до 90+ мин).
