= Кросс-валидация JSSE ↔ OpenSSL
:source-highlighter: coderay

== Цель

Проверить, что реализация JSSE (https://docs.oracle.com/en/java/javase/21/security/java-secure-socket-extension-jsse-reference-guide.html[JSSE API])
библиотеки crypto-gost корректно взаимодействует с эталонной реализацией OpenSSL
(патченая сборка 3.6 с gostprov) на уровне:

* `GostSSLSocket` / `GostSSLServerSocket` (InputStream/OutputStream API)
* `GostSSLEngine` (wrap/unwrap ByteBuffer API)

Оба API должны быть совместимы с OpenSSL `s_client` и `s_server` для TLS 1.3 + ГОСТ.

== Методология

=== Серверные тесты

crypto-gost выступает в роли сервера, OpenSSL `s_client` — в роли клиента.

. crypto-gost создаёт `GostSSLEngine` (серверный режим) или `GostSSLServerSocket`
. Открывает TCP-порт, ждёт подключения
. `s_client` подключается, выполняет TLS 1.3 handshake
. `s_client` отправляет `GET / HTTP/1.0\nHost: example.com\n\n`
. Сервер отвечает `HTTP/1.1 200 OK\n...INTEROP_OK`
. Проверяется, что `s_client` получил строку `INTEROP_OK`

=== Клиентские тесты

OpenSSL `s_server` — в роли сервера, crypto-gost — в роли клиента.

. `s_server` запускается с ГОСТ-сертификатом на случайном порту
. crypto-gost создаёт `GostSSLEngine` (клиентский режим) или `GostSSLSocket`
. Выполняется TLS 1.3 handshake
. crypto-gost отправляет `GET / HTTP/1.0\nHost: localhost\n\n`
. Проверяется, что ответ содержит информацию о соединении

== Тестовые сценарии

=== Engine-сервер (GostSSLEngine + wrap/unwrap + TCP)

Ручной цикл wrap/unwrap через ServerSocket — эмуляция неблокирующего IO.

|===
| Тест | Сценарий | Кривые | Suite |

| roundtrip L | GET → INTEROP_OK | GC256B | L |
| roundtrip S | GET → INTEROP_OK | GC256B | S |
| все кривые | handshake | GC256B, GC512A, GC512B | L |
| mTLS | взаимная аутентификация | GC256B | L |
| payload | 1, 15, 16, 1024, 12000 байт | GC256B | L |
| group mismatch | сервер GC512C, клиент GC256B | fallback | L |
| KeyUpdate | initiateKeyUpdate() после handshake | GC256B | L |
|===

=== Engine-клиент (s_server + GostSSLEngine)

|===
| Тест | Сценарий | Кривые | Suite |

| roundtrip L | GET → ответ | GC256B | L |
| roundtrip S | GET → ответ | GC256B | S |
| все кривые | handshake | GC256B, GC512A, GC512B | L |
|===

=== Socket-сервер (GostSSLServerSocket + s_client)

|===
| Тест | Сценарий | Кривые | Suite |

| roundtrip L | GET → INTEROP_OK | GC256B | L |
| roundtrip S | GET → INTEROP_OK | GC256B | S |
| mTLS | взаимная аутентификация | GC256B | L |
|===

=== Socket-клиент (s_server + GostSSLSocket)

|===
| Тест | Сценарий | Кривые | Suite |

| roundtrip L | GET → ответ | GC256B | L |
| roundtrip S | GET → ответ | GC256B | S |
|===

== Покрытие

* 23 тестовых сценария (22 активных + 1 @Disabled KeyUpdate)
* 3 кривые: GC256B (CryptoPro-A), GC512A (TC26-512A), GC512B (TC26-512B)
* 2 cipher suites: L и S
* 2 API: SSLEngine и SSLSocket
* 5 размеров payload
* mTLS
* Group mismatch (fallback)
* KeyUpdate (@Disabled до подтверждения OpenSSL)

== Требования к окружению

* OpenSSL 3.6+ с gostprov (патченая сборка)
* GOST TLS 1.3 cipher suites: `TLS_GOSTR341112_256_WITH_KUZNYECHIK_MGM_L` (0xC103) и `TLS_GOSTR341112_256_WITH_KUZNYECHIK_MGM_S` (0xC105)
* Java 21+ (virtual threads)
* Maven 3.8+
